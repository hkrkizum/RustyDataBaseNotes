# UX / Interaction Checklist: テーブルビュー操作拡張

**Purpose**: ソート・フィルタ・グルーピングの UI インタラクション，視覚フィードバック，操作導線，アクセシビリティに関する仕様の品質を検証する
**Created**: 2026-03-22
**Feature**: [spec.md](../spec.md)

## ソートインタラクションの仕様明確性

- [ ] CHK001 - FR-001 のソートインジケータは「▲▼等」と例示されているが，具体的なインジケータの視覚仕様（アイコン，色，サイズ）は定義されているか，それとも実装裁量に委ねられることが明示されているか？ [Clarity, Spec §FR-001]
- [ ] CHK002 - カラムヘッダーのソート操作可能であることのアフォーダンス（hover 状態，カーソル変化等）の要件は定義されているか？ [Gap]
- [ ] CHK003 - US1 シナリオ 3 でソート解除後に「デフォルトの表示順（作成日時順）」に戻るとされているが，ソート解除のインジケータ消失タイミング（即座 / アニメーション付き）は定義されているか？ [Gap, Spec §US1]
- [ ] CHK004 - US3 でカラムヘッダークリックが「既存の複数ソートをクリアし単一ソートに切り替え」る動作は，ユーザーが意図せず複数ソート設定を失うリスクへの対処（確認ダイアログ等）の要件が定義されているか？ [Gap, Spec §US3]
- [ ] CHK005 - FR-003 の「ソート設定パネル」のアクセス方法（ボタン位置，開閉方法）は仕様で定義されているか？ [Gap, Spec §FR-003]

## フィルタインタラクションの仕様明確性

- [ ] CHK006 - フィルタ条件の追加 UI フロー（プロパティ選択 → 演算子選択 → 値入力）の順序と各ステップの要件は定義されているか？ [Gap]
- [ ] CHK007 - フィルタの演算子選択 UI で，プロパティ型に応じて選択可能な演算子が動的に変わる要件は FR-004 で演算子リストが定義されているが，UI 上での制約表現（非対応演算子の非表示 / disabled 表示）は定義されているか？ [Gap, Spec §FR-004]
- [ ] CHK008 - FR-005 の「フィルタ条件の追加・編集・個別削除が可能」で，「編集」が演算子の変更・値の変更・プロパティの変更のどれを含むかが明示されているか？ [Clarity, Spec §FR-005]
- [ ] CHK009 - フィルタ条件の個別削除 UI（削除ボタン，スワイプ，ドラッグ等）の要件は定義されているか？ [Gap, Spec §FR-005]
- [ ] CHK010 - フィルタが適用中であることを示す「インジケータ」（US2 冒頭）の具体的な表示位置・視覚仕様は定義されているか？ [Clarity, Spec §US2]
- [ ] CHK011 - FR-010 の 0 件表示時の「フィルタの解除手段」が具体的に何を指す（全解除ボタン / 個別条件の削除リンク / フィルタパネルへのリンク）かは定義されているか？ [Ambiguity, Spec §FR-010]

## グルーピングインタラクションの仕様明確性

- [ ] CHK012 - FR-006 のグループヘッダーの視覚仕様（背景色の区別，セパレータ，インデント等）は定義されているか，それとも実装裁量に委ねられることが明示されているか？ [Gap, Spec §FR-006]
- [ ] CHK013 - グループの折りたたみ・展開操作のインタラクションパターン（ヘッダー全体のタップ / 折りたたみアイコン / キーボードショートカット）は定義されているか？ [Gap, Spec §FR-006]
- [ ] CHK014 - グルーピング設定の UI アクセス方法（メニュー位置，プロパティ選択 UI）の要件は定義されているか？ [Gap]
- [ ] CHK015 - グルーピング解除の操作方法（パネル内の解除ボタン / ヘッダーメニュー等）は定義されているか？ [Gap]
- [ ] CHK016 - 「未設定」グループのヘッダー表示ラベルの要件（「未設定」固定か，カスタマイズ可能か）は定義されているか？ [Clarity, Spec §FR-006]

## パネル・モーダルの操作設計

- [ ] CHK017 - ソート設定パネル（FR-003），フィルタパネル，グルーピングパネルの 3 つの設定 UI が同時に開けるか排他的かの要件は定義されているか？ [Gap]
- [ ] CHK018 - 各設定パネルの開閉方法（トグルボタン / ドロップダウン / サイドパネル / モーダル）の要件は定義されているか？ [Gap]
- [ ] CHK019 - 設定パネルでの変更が即座に反映されるか（ライブプレビュー），適用ボタンで確定するかの要件は定義されているか？ [Gap]
- [ ] CHK020 - FR-003 でソート条件の「優先順位変更」の UI 操作方法（ドラッグ＆ドロップ / 上下ボタン / 数値入力）は定義されているか？ [Gap, Spec §FR-003]

## リセット・一括操作の仕様

- [ ] CHK021 - FR-008 の「設定をリセット」操作の UI 配置（テーブルビュー上のボタン / 各パネル内 / コンテキストメニュー）は定義されているか？ [Gap, Spec §FR-008]
- [ ] CHK022 - リセット操作が不可逆であることへの確認 UI（確認ダイアログ等）の要件は定義されているか？ [Gap, Spec §FR-008]
- [ ] CHK023 - ソート・フィルタ・グルーピングのうち一部のみリセットする要件は FR-008 の「一括リセット」とは別に定義されているか？（例: フィルタのみクリア） [Gap, Spec §FR-008]

## 視覚フィードバックと状態表示

- [ ] CHK024 - ソート・フィルタ・グルーピングの各設定状態を一目で把握できるサマリ表示（例: ツールバーにアクティブな条件数を表示）の要件は定義されているか？ [Gap]
- [ ] CHK025 - フィルタ適用中にページを新規追加した場合（FR-011），新規ページがフィルタ条件を満たさずテーブルに表示されない状況での視覚フィードバック要件は定義されているか？ [Gap, Spec §FR-011]
- [ ] CHK026 - ソート・フィルタ・グルーピング操作中のローディング状態（操作後〜結果反映までの間）の UI 要件は定義されているか？ [Gap]

- [ ] CHK028 - 各設定パネルのキーボード操作（Tab 移動，Enter 確定，Escape 閉じ）の要件は定義されているか？ [Gap]

## 受入シナリオの操作フロー網羅性

- [ ] CHK030 - US1〜US4 の受入シナリオはすべて Given/When/Then 形式だが，ソート設定パネルの操作フロー（US3: 条件追加 → 優先順位変更 → 条件削除）の詳細ステップは十分に具体的か？ [Clarity, Spec §US3]
- [ ] CHK031 - US2 の受入シナリオにフィルタ条件の「編集」操作（既存条件の値変更等）のシナリオが含まれていないが，FR-005 の「編集」要件との整合性は確保されているか？ [Consistency, Spec §US2 / §FR-005]
- [ ] CHK032 - フィルタ・ソート・グルーピングの 3 機能を同時に操作する複合シナリオ（例: グルーピング中にフィルタを追加してソートを変更）の受入シナリオは定義されているか？ [Coverage, Gap]
