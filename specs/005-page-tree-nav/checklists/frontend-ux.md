**Depth**: Standard | **Audience**: レビュアー（PR）

## Requirement Completeness

- [x] CHK008 - サイドバーでアクティブなアイテムがスクロール範囲外にある場合（ナビゲーション後やアプリ起動時の復元時）、自動スクロールでビューポート内に表示する要件はあるか？ <!-- spec.md FR-002 に「アクティブアイテムがスクロール範囲外にある場合は自動的にビューポート内にスクロールする」と明記 -->

## Requirement Clarity

- [x] CHK010 - FR-013 の新規作成ボタンの「種別選択 UI（ドロップダウン等）」の「等」は何を指すか？ドロップダウン以外の選択肢（ポップオーバー・モーダル等）が許容されるのか、ドロップダウンに確定しているのか？ <!-- spec.md FR-013 に「種別選択 UI（ドロップダウンメニュー）」と確定済み。「等」は除去されている -->
- [x] CHK011 - FR-007 の「挿入位置インジケーターライン」はドロップ先アイテムの上・下・間のどこに表示されるか？同一親内での並び替え操作を本スコープでは行わない前提で、ラインの意味（「この親の子になる」vs「この位置に挿入する」）は明確か？ <!-- spec.md FR-007 に「ドロップ先アイテムの子になることを意味する」と明記。plan.md にルートレベル用インジケーターも定義 -->
- [x] CHK012 - 自動保存の「debounce 付き」のデバウンス間隔（ms）は定義されているか？ <!-- plan.md 自動保存パラメータに「デバウンス間隔: 500ms」と明記 -->
- [x] CHK013 - 自動保存失敗時の toast 警告の内容（メッセージテキスト・表示持続時間・ユーザーアクション）は定義されているか？リトライ回数（最大3回）のインターバルは定義されているか？ <!-- plan.md 自動保存パラメータに「指数バックオフ（1s → 2s → 4s）」「Toast:『保存に失敗しました』表示時間5秒，自動消去」と明記 -->
- [x] CHK014 - FR-010 サイドバー非表示時のメインコンテンツ領域のレイアウト変化（フル幅展開するか・余白が残るか）は定義されているか？ <!-- plan.md サイドバーレイアウトに「メインコンテンツがフル幅に展開する。CSS Flexbox で動的に制御」と明記 -->

## Requirement Consistency

- [x] CHK015 - FR-011 でルートレベルの表示順は「created_at DESC」と定義されているが、同一親内の子ページの表示順も同じルール（created_at DESC）であるか明示されているか？ <!-- spec.md FR-011 に「ルートレベルおよび同一親内の子ページは…created_at DESC（作成日時降順）で表示する」と明記 -->
- [x] CHK016 - FR-013 の「新規作成後に自動遷移＋インライン名前編集を即時開始」と、FR-017 の「最後に開いたアイテムの localStorage 永続化」の相互作用は定義されているか？新規作成アイテムが即座に last-opened-item として保存されるタイミングは明確か？ <!-- spec.md FR-013 に「新規作成後は作成されたアイテムに自動遷移し（同時に last-opened-item を更新）」と明記 -->
- [x] CHK017 - コンテキストメニューの「削除」アクション実行時に確認ダイアログ（子ページ昇格の説明等）を表示する要件はあるか？特に子ページを持つ親の削除時、ユーザーへの事前通知は定義されているか？ <!-- spec.md FR-013 に「『削除』実行前に確認ダイアログを表示する。子ページを持つ親の削除時は昇格される旨を確認ダイアログ内で説明する」と明記 -->
- [x] CHK018 - FR-007 の D&D 操作と FR-013 のコンテキストメニュー操作の排他性は定義されているか？ D&D 中にコンテキストメニューが開かれた場合の動作は規定されているか？ <!-- plan.md ドラッグ＆ドロップ詳細に「D&D 中（isDragging 状態）はコンテキストメニューの表示を無効化する」と明記 -->
