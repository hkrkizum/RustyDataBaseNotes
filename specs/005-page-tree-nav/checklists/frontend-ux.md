# Frontend/UX Requirements Quality Checklist: Page Tree Navigation

**Purpose**: サイドバーUI・ツリーインタラクション・D&D・インライン編集・状態管理・localStorage永続化のフロントエンド要件の完全性・明確性・一貫性を検証する
**Created**: 2026-03-22
**Feature**: [spec.md](../spec.md) | [plan.md](../plan.md) | [contracts/ipc-commands.md](../contracts/ipc-commands.md)
**Focus**: フロントエンド/UX（サイドバーUI・ツリーD&D・インライン編集・状態管理）
**Depth**: Standard | **Audience**: レビュアー（PR）

## Requirement Completeness

- [ ] CHK001 - サイドバーの固定幅が「240–260px」と範囲で記載されているが、実装時に使用する具体的な値は決定されているか？ [Clarity, Spec §FR-010]
- [ ] CHK002 - ツリーの各階層レベルにおけるインデント量（px または spacing token）は定義されているか？5階層ネスト時にサイドバー固定幅内でコンテンツが視認可能であることの要件はあるか？ [Gap, Spec §FR-006]
- [ ] CHK003 - サイドバー内でページタイトルが固定幅を超過する場合の表示処理（トランケーション・省略記号・ツールチップ等）は定義されているか？ [Gap]
- [ ] CHK004 - D&D 操作時にドラッグ対象がサイドバーの上端/下端に近づいた場合の自動スクロール動作は定義されているか？ [Gap, Spec §FR-007]
- [ ] CHK005 - コンテキストメニュー「...」ボタンのホバー時表示について、表示タイミング（遅延の有無）・表示位置・ボタンサイズは定義されているか？ [Gap, Spec §FR-013]
- [ ] CHK006 - インライン名前編集（FR-013）のテキスト入力フィールドのバリデーション UX（空文字入力時・255文字超過時）は定義されているか？spec の PageTitle 制約（1-255文字）に対応するフロントエンド側の振る舞いは明示されているか？ [Gap, Spec §FR-013]
- [ ] CHK007 - サイドバーのスクロール動作（オーバーフロー時のスクロールバー表示・スクロール領域の範囲）は定義されているか？ヘッダー・新規作成ボタンはスクロール対象外（固定表示）か？ [Gap]
- [ ] CHK008 - サイドバーでアクティブなアイテムがスクロール範囲外にある場合（ナビゲーション後やアプリ起動時の復元時）、自動スクロールでビューポート内に表示する要件はあるか？ [Gap, Spec §FR-002]

## Requirement Clarity

- [ ] CHK009 - FR-002 のアクティブアイテムの「背景色ハイライト（shadcn/ui 標準スタイル）」は、どの shadcn/ui コンポーネントのどの状態を指すか明確か？フォーカス状態・選択状態の区別は定義されているか？ [Ambiguity, Spec §FR-002]
- [ ] CHK010 - FR-013 の新規作成ボタンの「種別選択 UI（ドロップダウン等）」の「等」は何を指すか？ドロップダウン以外の選択肢（ポップオーバー・モーダル等）が許容されるのか、ドロップダウンに確定しているのか？ [Ambiguity, Spec §FR-013]
- [ ] CHK011 - FR-007 の「挿入位置インジケーターライン」はドロップ先アイテムの上・下・間のどこに表示されるか？同一親内での並び替え操作を本スコープでは行わない前提で、ラインの意味（「この親の子になる」vs「この位置に挿入する」）は明確か？ [Ambiguity, Spec §FR-007]
- [ ] CHK012 - 自動保存の「debounce 付き」のデバウンス間隔（ms）は定義されているか？ [Clarity, Assumptions §自動保存]
- [ ] CHK013 - 自動保存失敗時の toast 警告の内容（メッセージテキスト・表示持続時間・ユーザーアクション）は定義されているか？リトライ回数（最大3回）のインターバルは定義されているか？ [Clarity, Clarifications §自動保存]
- [ ] CHK014 - FR-010 サイドバー非表示時のメインコンテンツ領域のレイアウト変化（フル幅展開するか・余白が残るか）は定義されているか？ [Gap, Spec §FR-010]

## Requirement Consistency

- [ ] CHK015 - FR-011 でルートレベルの表示順は「created_at DESC」と定義されているが、同一親内の子ページの表示順も同じルール（created_at DESC）であるか明示されているか？ [Consistency, Spec §FR-011]
- [ ] CHK016 - FR-013 の「新規作成後に自動遷移＋インライン名前編集を即時開始」と、FR-017 の「最後に開いたアイテムの localStorage 永続化」の相互作用は定義されているか？新規作成アイテムが即座に last-opened-item として保存されるタイミングは明確か？ [Consistency, Spec §FR-013 / FR-017]
- [ ] CHK017 - コンテキストメニューの「削除」アクション実行時に確認ダイアログ（子ページ昇格の説明等）を表示する要件はあるか？特に子ページを持つ親の削除時、ユーザーへの事前通知は定義されているか？ [Gap, Spec §FR-008 / FR-013]
- [ ] CHK018 - FR-007 の D&D 操作と FR-013 のコンテキストメニュー操作の排他性は定義されているか？ D&D 中にコンテキストメニューが開かれた場合の動作は規定されているか？ [Gap]

## Scenario Coverage

- [ ] CHK019 - サイドバーのデータ取得タイミング（アプリ起動時のみ？画面遷移のたびに？バックグラウンドポーリング？）と、バックエンド操作（作成・移動・削除・名前変更）後のサイドバー更新戦略（楽観的更新 vs 再取得）は定義されているか？ [Gap]
- [ ] CHK020 - D&D でページをルートレベルに昇格させる操作の UI インタラクション（ドロップ先としてルートレベルをどのように示すか）は定義されているか？ [Gap, Spec §FR-007]
- [ ] CHK021 - サイドバー表示/非表示トグル（Cmd/Ctrl+B）がテキスト入力中（インライン名前編集中・エディタ編集中）に押された場合の動作は定義されているか？ [Gap, Spec §FR-010]
- [ ] CHK022 - FR-017 の起動時復元で、最後に開いたアイテムがツリー内の折りたたまれた子ページだった場合、その祖先ノードを自動展開する要件はあるか？ [Gap, Spec §FR-014 / FR-017]
- [ ] CHK023 - データベースグループ内のページをクリックしてエディタで開く場合と、スタンドアロンページをクリックする場合で、画面遷移先やエディタの動作に差異がないことは明示されているか？ [Consistency, Spec §FR-002 / FR-011]

## Edge Case Coverage

- [ ] CHK024 - 5階層目（最大深度）にあるページのコンテキストメニューに「子ページ作成」が表示されるか？表示される場合、クリック時のエラー UX（MaxDepthExceeded）は定義されているか？非表示にする場合、その要件は定義されているか？ [Gap, Spec §FR-004 / FR-013]
- [ ] CHK025 - D&D でアイテムを自分自身の上にドロップした場合の視覚フィードバック（禁止カーソルか no-op か）は定義されているか？ [Gap, Spec §FR-007]
- [ ] CHK026 - インライン名前編集中にサイドバー外をクリックした場合の動作（確定？キャンセル？）は定義されているか？Enter/Esc 以外の確定・キャンセル手段は規定されているか？ [Gap, Spec §FR-013]
- [ ] CHK027 - localStorage のデータ（展開状態・表示状態・最後に開いたアイテム）が破損・不正値の場合のフォールバック動作は定義されているか？ [Gap, Spec §FR-014 / FR-017]
- [ ] CHK028 - ページもデータベースも存在しない空状態から新規作成した直後の一連のフロー（作成→自動遷移→インライン名前編集）は US2 Acceptance Scenarios でカバーされているか？ [Coverage, Spec §Edge Cases / FR-013]

## Acceptance Criteria Quality

- [ ] CHK029 - CC-003 のフロントエンド側パフォーマンス目標（ツリー展開/折りたたみ ≤50ms、サイドバークリック→画面遷移 ≤100ms）の計測方法・計測ポイント（ユーザーイベント発火からレンダリング完了まで等）は定義されているか？ [Measurability, Spec §CC-003]
- [ ] CHK030 - SC-005「ビジュアルレビューで色・タイポグラフィ・スペーシングの一貫性が確認される」の具体的なレビュー基準（どの画面をどの観点で比較するか）は定義されているか？ [Measurability, Spec §SC-005]
