**Purpose**: ソート・フィルタ・グルーピングの UI インタラクション，視覚フィードバック，操作導線，アクセシビリティに関する仕様の品質を検証する
**Created**: 2026-03-22
**Feature**: [spec.md](../spec.md)

- [x] CHK003 - US1 シナリオ 3 でソート解除後に「デフォルトの表示順（作成日時順）」に戻るとされているが，ソート解除のインジケータ消失タイミング（即座 / アニメーション付き）は定義されているか？ <!-- spec.md Assumptions にて「UI 状態変化のアニメーション・トランジションは本スコープでは定義しない。インジケータの表示・非表示は即時切り替えとする」と明記。即時切り替え方式が確定済み -->
- [x] CHK004 - US3 でカラムヘッダークリックが「既存の複数ソートをクリアし単一ソートに切り替え」る動作は，ユーザーが意図せず複数ソート設定を失うリスクへの対処（確認ダイアログ等）の要件が定義されているか？ <!-- spec.md Assumptions にて「ソート・フィルタ・グルーピング操作において確認ダイアログは表示しない。先行事例（Notion 等）に倣い，操作の即時反映を優先する。Undo 機能は後続スコープとする」と明記。確認ダイアログを設けない設計判断が明示されている -->
- [x] CHK005 - FR-003 の「ソート設定パネル」のアクセス方法（ボタン位置，開閉方法）は仕様で定義されているか？ <!-- spec.md「UI Interaction Specification > パネルアクセスと操作方式」にて「テーブルビュー上部のツールバー領域からアクセスする。各パネルはドロップダウン方式で開閉する」「パネル外をクリックするとパネルが閉じる」と明記 -->

- [x] CHK006 - フィルタ条件の追加 UI フロー（プロパティ選択 → 演算子選択 → 値入力）の順序と各ステップの要件は定義されているか？ <!-- spec.md「UI Interaction Specification > フィルタパネル」にて「フィルタ条件の追加: プロパティ選択 → 演算子選択 → 値入力の 3 ステップ」と明記。順序と各ステップが定義されている -->
- [x] CHK007 - フィルタの演算子選択 UI で，プロパティ型に応じて選択可能な演算子が動的に変わる要件は FR-004 で演算子リストが定義されているが，UI 上での制約表現（非対応演算子の非表示 / disabled 表示）は定義されているか？ <!-- spec.md「UI Interaction Specification > フィルタパネル」にて「演算子リストはプロパティ型に応じて動的にフィルタリングされる（FR-004 に非対応の演算子は選択肢に表示しない）」と明記。非表示方式であることが確定 -->
- [x] CHK008 - FR-005 の「フィルタ条件の追加・編集・個別削除が可能」で，「編集」が演算子の変更・値の変更・プロパティの変更のどれを含むかが明示されているか？ <!-- spec.md「UI Interaction Specification > フィルタパネル」にて「フィルタ条件の編集: 既存条件の演算子・値・対象プロパティを直接変更可能」と明記。演算子・値・プロパティの 3 要素すべてが編集対象として定義済み -->
- [x] CHK009 - フィルタ条件の個別削除 UI（削除ボタン，スワイプ，ドラッグ等）の要件は定義されているか？ <!-- spec.md「UI Interaction Specification > フィルタパネル」にて「フィルタ条件の個別削除: 各条件行の削除ボタン」と明記 -->
- [x] CHK010 - フィルタが適用中であることを示す「インジケータ」（US2 冒頭）の具体的な表示位置・視覚仕様は定義されているか？ <!-- spec.md「UI Interaction Specification > 視覚フィードバック」にて「フィルタ適用中: ツールバーのフィルタボタンにアクティブ状態を表示（例: 色変更，バッジでの条件数表示）。具体的な視覚表現は実装裁量に委ねる」と明記。表示位置（ツールバーのフィルタボタン）と表現方法（アクティブ状態 + 例示）が定義済み -->
- [x] CHK011 - FR-010 の 0 件表示時の「フィルタの解除手段」が具体的に何を指す（全解除ボタン / 個別条件の削除リンク / フィルタパネルへのリンク）かは定義されているか？ <!-- spec.md「視覚フィードバック」にて「フィルタ適用中に表示件数が 0 件の場合: テーブル領域に『条件に一致するページがありません』メッセージと『すべてのフィルタを解除』ボタンを表示する」と明記。具体的な UI 形態（メッセージ＋全解除ボタン）が定義済み -->

- [ ] CHK025 - フィルタ適用中にページを新規追加した場合（FR-011），新規ページがフィルタ条件を満たさずテーブルに表示されない状況での視覚フィードバック要件は定義されているか？ [Partial] <!-- FR-011 で動作仕様（非表示だがフィルタ解除で表示）は明確。Edge Cases にも記載あり。フィルタアクティブ状態の常時表示と 0 件時のメッセージ＋解除ボタンは定義済み。ただし新規追加ページが非表示になった際の個別通知（例: トースト「追加されたページはフィルタにより非表示です」等）は未定義 -->
- [x] CHK026 - ソート・フィルタ・グルーピング操作中のローディング状態（操作後〜結果反映までの間）の UI 要件は定義されているか？ <!-- spec.md「視覚フィードバック」にて「CC-003 のパフォーマンス目標（500ms）を超える応答が見込まれる場合（1,000 件超等），テーブル領域にローディングインジケータを表示する（SHOULD）」と明記。ローディング状態の表示条件と配置場所が定義済み -->
